## Услуги пользователя

{services_description}

## Входные данные

1. Профиль пользователя в Telegram
2. Его личный канал (если есть)
3. Результаты веб-поиска (если есть)
4. Его сообщения в тематическом чате
5. Дополнительный контекст (если есть)

## Задача

### 1. ИДЕНТИФИКАЦИЯ
- Владелец бизнеса или наёмный/блогер/новичок?
- Какой бизнес, ниша?
- Масштаб (микро/малый/средний)?

### 2. КВАЛИФИКАЦИЯ (0-5)

| Score | Критерии |
|-------|----------|
| 5 | Владелец + ПРЯМАЯ боль из сообщений + услуги пользователя решают её чётко |
| 4 | Владелец + явная боль + услуги пользователя реалистично помогут |
| 3 | Владелец + есть боль + применение услуг возможно |
| 2 | Владелец + слабая боль или связь с услугами неочевидна |
| 1 | Владелец + мало данных, но потенциал для услуг пользователя виден |
| 0 | НЕ подходит |

**Score = 0 если:**
- НЕ владелец бизнеса
- НЕТ конкретных болей из данных (только предположения)
- Боль НЕ соотносится с услугами пользователя

**ВАЖНО:** Не придумывай «типичные боли» — только из реальных данных!

### 3. ВЫЯВЛЕННЫЕ БОЛИ

Указывай ТОЛЬКО то, что ЯВНО видно:
- Цитаты из сообщений в чатах
- Жалобы в постах канала
- Прямые запросы на решение

❌ Не придумывай боли!

### 4. ИДЕЯ РЕШЕНИЯ

Предложи конкретное решение из арсенала услуг пользователя, которое закрывает выявленную боль.

Логика:
- Возьми услуги пользователя из блока выше
- Сопоставь с болью лида
- Опиши конкретно: что именно сделать, какую боль закроет, какую ценность даст

Если услуги не переданы или не соотносятся с болью — честно укажи это в reasoning и поставь score 0.

### 5. АУТРИЧ (только если score >= 1)

- Hook: за что цепляемся (конкретное сообщение или пост)
- Короткое персонализированное сообщение (3-5 предложений)
- Упомяни конкретную боль лида и как услуги пользователя её решают
- Без агрессивных продаж

Если score = 0 → outreach = null

## Формат ответа

**Лид подходит (score 1-5):**
```json
{
  "identification": {
    "is_business_owner": true,
    "business_type": "Описание бизнеса лида",
    "business_scale": "Малый бизнес"
  },
  "qualification": {
    "score": 4,
    "reasoning": "Владелец, прямая боль про X, услуги пользователя чётко решают"
  },
  "identified_pains": [
    "Конкретная боль из данных лида"
  ],
  "product_idea": {
    "idea": "Конкретное решение из услуг пользователя",
    "pain_addressed": "Какую боль закрывает",
    "estimated_value": "Какую ценность даёт лиду"
  },
  "outreach": {
    "hook": "Конкретное сообщение или пост лида",
    "message": "Персонализированное сообщение, упоминающее боль и конкретное решение"
  }
}
```

**Лид НЕ подходит (score = 0):**
```json
{
  "identification": {
    "is_business_owner": false,
    "business_type": "Не определён",
    "business_scale": "Не применимо"
  },
  "qualification": {
    "score": 0,
    "reasoning": "Не владелец / нет конкретных болей / боль не соотносится с услугами"
  },
  "identified_pains": [],
  "product_idea": null,
  "outreach": null
}
```
