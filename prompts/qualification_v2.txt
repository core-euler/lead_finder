Контекст: Анализируем потенциального клиента для агентства,
которое разрабатывает Telegram-ботов и Mini Apps.

Доступные данные:
1. Профиль пользователя в Telegram
2. Его личный канал (если есть)
3. Результаты веб-поиска (сайт, соцсети, упоминания)
4. Его сообщения в тематическом чате (если есть)

Задача:

1. ИДЕНТИФИКАЦИЯ
   - Это владелец бизнеса или наёмный сотрудник/блогер?
   - Какой бизнес (ниша, продукт)?
   - Какой масштаб (оборот, команда)?

2. КВАЛИФИКАЦИЯ (1-10)
   ВАЖНО: Оценка основана на трёх факторах:
   1) Владелец бизнеса или нет
   2) Есть ли КОНКРЕТНЫЕ боли из сообщений/постов
   3) Можем ли МЫ РЕАЛЬНО решить эту боль Telegram-ботом

   - 9-10: Владелец + ПРЯМЫЕ боли из сообщений + МЫ МОЖЕМ сделать бот для этой боли
   - 7-8: Владелец + есть намеки на боли + решение возможно, но не очевидно
   - 5-6: Владелец + мало активности ИЛИ боли не очевидны ИЛИ решение сложное/нереалистичное
   - 3-4: Не ясно владелец или нет, либо нет релевантных болей для автоматизации
   - 1-2: Не подходит (блогер, наёмный, неактивен, нет болей)

   КРИТЕРИИ:
   - Если нет КОНКРЕТНЫХ болей из реальных сообщений/постов - максимум 6 баллов
   - Если пишешь "боли предполагаемые", "нет прямых болей", "типичные боли" - это 5-6 максимум
   - Если боль есть, но МЫ НЕ МОЖЕМ сделать бот (нужна интеграция с API, которого нет) - максимум 6 баллов

3. ВЫЯВЛЕННЫЕ БОЛИ
   ВАЖНО: Указывай ТОЛЬКО боли, которые РЕАЛЬНО видны в данных!

   - Прямые (из сообщений в чате, постов) - ПРИОРИТЕТ
   - Цитаты из сообщений с болями (обязательно если есть)

   ❌ НЕ ПРИДУМЫВАЙ "типичные для бизнеса" боли, если их нет в данных!
   Если боли нет - так и пиши: "Конкретных болей не выявлено"

4. ИДЕЯ РЕШЕНИЯ
   КРИТИЧЕСКИ ВАЖНО: Предлагай ТОЛЬКО то, что МЫ РЕАЛЬНО МОЖЕМ СДЕЛАТЬ!

   Если боль требует интеграции с API, которого нет - это НЕ наш клиент!

   ✅ МЫ МОЖЕМ (реалистичные боты):
   - Приём заявок/обращений через форму в боте
   - Консультации и FAQ (с AI или кнопками)
   - Уведомления о новых событиях (через вебхуки или периодические проверки)
   - Калькуляторы и расчёты на основе данных от пользователя
   - Опросы и сбор обратной связи
   - Каталог товаров/услуг с описаниями
   - Запись на услуги/встречи (календарь в боте)
   - Рассылки и контент-доставка подписчикам
   - Простые CRM-интеграции (если есть webhook/API у CRM)
   - Напоминания о событиях/дедлайнах (на основе данных от пользователя)

   ❌ МЫ НЕ МОЖЕМ (нереалистичные решения):
   - Автоматическое отслеживание выплат/штрафов Wildberries/Ozon (нет открытого API)
   - Мониторинг налоговых статусов и ИП (нет интеграции с ФНС)
   - Автопарсинг данных с защищённых платформ без API
   - Прямые изменения данных в системах без API (например, изменение товаров на WB)
   - Автоматическое отслеживание ошибок склада маркетплейса (нет API)

   ПРАВИЛО: Бот решает проблему через КОММУНИКАЦИЮ, СБОР ДАННЫХ от пользователя, НАПОМИНАНИЯ.
   Если для решения нужно "интегрироваться с платформой X" (а API нет) - это НЕ наш клиент.

   В ответе укажи:
   - Конкретный бот/Mini App (что именно делает)
   - Какую боль закрывает (из тех, что ты выявил)
   - Реалистично ли это решение (можем ли мы это сделать БЕЗ недоступных API)

5. ПЕРСОНАЛИЗАЦИЯ АУТРИЧА
   - На что ссылаться (пост, сообщение, статья)
   - Какой hook использовать
   - Готовое первое сообщение

**Формат вывода:**
Верни результат в виде строгого JSON объекта, следующего схеме:
```json

  "identification": {

  "qualification": {
    "score": 8, // Оценка от 1 до 10 (без бонусов, чисто на основе данных)
```
